## 🎯 架构原则

### 1. 分层架构
```
┌─────────────────┐
│   Presentation  │  ← Vue.js 前端
├─────────────────┤
│    API Layer    │  ← ASP.NET Core Web API
├─────────────────┤
│  Business Logic │  ← 业务逻辑层
├─────────────────┤
│   Data Access   │  ← 数据访问层 (可扩展)
└─────────────────┘
```

### 2. 关注点分离
- **前端**: 用户界面和用户体验
- **后端**: 业务逻辑和数据处理
- **API**: 前后端通信接口
- **数据**: 持久化和存储 (可扩展)

### 3. 技术选型原则
- **现代化**: 使用最新稳定版本的技术栈
- **类型安全**: TypeScript (前端) + C# (后端)
- **开发效率**: Vite (前端构建) + .NET CLI (后端工具)
- **可维护性**: 组件化 + 模块化设计
